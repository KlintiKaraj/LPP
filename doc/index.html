Core Identity

What is LPP?

LPP (LPP++) is a modern scripting language built on top of the Lua runtime. It extends Lua's capabilities with enhanced standard libraries while maintaining full compatibility with Lua's syntax and semantics.

LPP is not a new interpreter from scratch—it is a modified Lua-based runtime that adds practical functionality for modern scripting needs. It keeps Lua's lightweight, fast, and embeddable nature while providing developers with ready-to-use tools for common tasks.

Key Features

  • HTTP Built-in HTTP client with GET, POST, and unified request API
  • File Extended file handling utilities
  • OS Enhanced operating system interaction tools
  • Syntax Modern syntax improvements including endb (end block) keyword
  • Cross-Platform Native support for Windows, Linux, and macOS
  • Production Structured error handling, timeouts, and robust API design

Vision and Goals

LPP aims to provide a scripting language that feels familiar to Lua developers while offering the tools needed for real-world applications. The philosophy is simple: extend, don't replace. By building on Lua's proven foundation, LPP delivers a stable, fast, and developer-friendly language for automation, scripting, and application development.

Installation

Prerequisites

  • GCC (MinGW on Windows) or compatible C compiler
  • Make (for Linux/macOS builds)
  • Basic build tools for your platform

Building from Source

Windows

# Clone the repository
git clone https://github.com/KlintiKaraj/LPP.git
cd lpp

# Build using the provided batch script
build.bat

# Install to PATH (run as Administrator)
install.ps1

Linux

# Clone the repository
git clone https://github.com/KlintiKaraj/LPP.git
cd lpp/src

# Build for Linux
make linux

# Install to system PATH
sudo ln -s ../bin/lpp /usr/local/bin/lpp

macOS

# Clone the repository
git clone https://github.com/KlintiKaraj/LPP.git
cd lpp/src

# Build for macOS
make macosx

# Install to system PATH
sudo ln -s ../bin/lpp /usr/local/bin/lpp

Running LPP

After installation, you can run LPP scripts from the command line:

lpp your_script.lpp

Or start an interactive REPL:

lpp

First Example

Create a file named hello.lpp:

print("Hello from LPP")

Run it:

lpp hello.lpp

Output:

Hello from LPP

Learn LPP

🧠 Basics

Print Output

The print() function outputs text to the console:

print("Hello, World!")
print("The answer is", 42)

Variables

Variables are dynamically typed and do not require declaration:

local name = "Alice"
local age = 25
local is_active = true

print(name, age, is_active)

Data Types

LPP supports all Lua data types:

-- Numbers
local x = 10
local y = 3.14

-- Strings
local text = "Hello"

-- Booleans
local flag = true

-- Tables (arrays/dictionaries)
local arr = {1, 2, 3}
local dict = {key = "value"}

-- Nil (absence of value)
local empty = nil

If Statements

Use then and endb for conditional blocks:

local x = 10

if x > 5 then
    print("big number")
endb

if x < 5 then
    print("small number")
else
    print("medium number")
endb

if x == 10 then
    print("exactly ten")
elseif x == 20 then
    print("exactly twenty")
else
    print("something else")
endb

Loops

While loop:

local i = 1
while i <= 5 do
    print(i)
    i = i + 1
endb

For loop:

for i = 1, 5 do
    print(i)
endb

-- Iterating over tables
local fruits = {"apple", "banana", "cherry"}
for i, fruit in ipairs(fruits) do
    print(i, fruit)
endb

Repeat until:

local i = 1
repeat
    print(i)
    i = i + 1
until i > 5

⚙️ Intermediate

Functions

Define functions using the function keyword and endb:

-- Basic function
function greet(name)
    print("Hello, " .. name)
endb

greet("World")

-- Function with return value
function add(a, b)
    return a + b
endb

local result = add(5, 3)
print(result)  -- 8

-- Anonymous functions
local multiply = function(a, b)
    return a * b
endb

print(multiply(4, 3))  -- 12

Tables / Objects

Tables are the primary data structure in LPP:

-- Array-style table
local colors = {"red", "green", "blue"}
print(colors[1])  -- red

-- Dictionary-style table
local person = {
    name = "Alice",
    age = 25,
    city = "Wonderland"
}
print(person.name)  -- Alice
print(person["age"])  -- 25

-- Nested tables
local company = {
    name = "TechCorp",
    employees = {
        {name = "Alice", role = "Engineer"},
        {name = "Bob", role = "Designer"}
    }
}
print(company.employees[1].name)  -- Alice

-- Adding to tables
local list = {}
list[1] = "first"
list["key"] = "value"

Basic Modular Structure

Organize code using modules:

-- mymodule.lpp
local mymodule = {}

function mymodule.hello()
    print("Hello from module")
endb

function mymodule.add(a, b)
    return a + b
endb

return mymodule
-- main.lpp
local mymodule = require("mymodule")

mymodule.hello()
print(mymodule.add(5, 3))

🌐 Standard Library

HTTP Requests

LPP provides a modern HTTP client with both simple and advanced APIs.

Simple GET:

local http = require("http")

local res, err = http.get("https://example.com")

if err then
    print("Error:", err)
else
    print("Status:", res.status)
    print("Body:", res.body)
endb

Simple POST:

local res, err = http.post("https://example.com/api", "key=value")

if err then
    print("Error:", err)
else
    print("Status:", res.status)
    print("Body:", res.body)
endb

Unified Request API:

local res, err = http.request({
    url = "https://example.com",
    method = "GET",
    headers = {
        ["Content-Type"] = "application/json"
    }
})

if err then
    print("Error:", err)
else
    print("Status:", res.status)
    print("Body:", res.body)
endb

File Handling

LPP provides extended file operations:

local file = require("file")

-- Read file
local content = file.read("path/to/file.txt")
print(content)

-- Write file
file.write("path/to/file.txt", "Hello, World!")

-- Check if file exists
if file.exists("path/to/file.txt") then
    print("File exists")
endb

OS Functions

Enhanced operating system interaction via os_pp library:

local os = require("os_pp")

-- Get current time
local time_val = os.time()
print(time_val)

-- Get date as table
local date_tbl = os.date()
print(date_tbl.year, date_tbl.month, date_tbl.day)

-- Get platform information
local platform = os.platform()
local arch = os.arch()
print("Platform:", platform)
print("Architecture:", arch)

-- Get process ID
local pid = os.pid()
print("PID:", pid)

-- Get current working directory
local cwd = os.cwd()
print("CWD:", cwd)

-- Get environment variable
local path = os.getenv("PATH")
print("PATH:", path)

-- Set environment variable
os.setenv("MY_VAR", "value")

-- Unset environment variable
os.unsetenv("MY_VAR")

-- Sleep for 1 second
os.sleep(1)

-- Get clock time
local clock1 = os.clock()
-- do some work
local clock2 = os.clock()
print("Elapsed:", clock2 - clock1)

-- Execute system command
local exit_code = os.run("ls -la")
print("Exit code:", exit_code)

Design Philosophy

Why LPP Exists

LPP was created to address a common problem: Lua is an excellent scripting language with a proven track record, but it lacks modern networking and system utilities out of the box. Developers often find themselves building the same HTTP clients, file handlers, and OS tools repeatedly across projects.

LPP solves this by providing these essential tools as part of the standard library, while maintaining Lua's core philosophy of simplicity and performance.

Why Lua Was Chosen as a Base

  • Proven Stability: Lua has been battle-tested for decades in production environments
  • Performance: One of the fastest scripting languages available
  • Simplicity: Clean, minimal syntax that's easy to learn and teach
  • Embeddable: Designed to be embedded in other applications
  • Community: Large ecosystem of libraries and knowledge

Why Minimal Syntax Changes

LPP intentionally keeps syntax changes minimal. The endb keyword is the primary syntax addition, replacing end for better readability in nested structures. This change:

  • Makes code more readable by explicitly marking "end of block"
  • Reduces cognitive load when scanning complex nested structures
  • Maintains compatibility with Lua's overall syntax philosophy

Why Features Are Added via Libraries

Instead of modifying the core virtual machine, LPP extends functionality through standard libraries:

  • Stability: Core VM remains unchanged, reducing bug surface
  • Modularity: Features can be loaded only when needed
  • Maintainability: Easier to update and debug individual libraries
  • Compatibility: Maintains compatibility with Lua's ecosystem

Stability and Simplicity Goals

LPP prioritizes stability and simplicity over feature bloat:

  • Each feature must solve a real, common problem
  • APIs should be intuitive and consistent
  • Documentation should be clear and comprehensive
  • Breaking changes should be avoided whenever possible
